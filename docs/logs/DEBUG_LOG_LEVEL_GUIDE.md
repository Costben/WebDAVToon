# 调试日志级别选择指南

## 日志级别说明

在设置页面的 **Debug** 卡片中，可以选择不同的日志级别来控制日志输出：

### 日志级别选项

1. **VERBOSE** (最详细)
   - 显示所有日志，包括最详细的调试信息
   - **适用场景**：深度调试，需要查看所有细节
   - **缺点**：日志文件会非常大，可能影响性能

2. **DEBUG** (推荐用于调试)
   - 显示 DEBUG 及以上级别的日志
   - **适用场景**：**日常调试和性能分析** ⭐
   - **包含内容**：
     - 图片加载过程（Bind、Preview、Compressed、FullImage）
     - 可见范围判断
     - 滚动位置信息
     - UI 显示时间点
     - 图片解码信息
   - **优点**：包含调试所需的关键信息，日志量适中

3. **INFO** (默认)
   - 显示 INFO 及以上级别的日志
   - **适用场景**：正常使用，查看重要事件
   - **包含内容**：
     - UI 可见时间点
     - 图片加载完成
     - 重要状态变化
   - **缺点**：缺少详细的调试信息

4. **WARN** (警告)
   - 只显示 WARN 和 ERROR 级别的日志
   - **适用场景**：只关注问题和警告
   - **缺点**：缺少调试信息

5. **ERROR** (错误)
   - 只显示 ERROR 级别的日志
   - **适用场景**：只关注错误
   - **缺点**：几乎没有调试信息

## 调试大图加载慢的问题时

### 推荐选择：**DEBUG** 级别

**原因**：
1. 包含图片加载的详细过程：
   - `[Bind]` - 图片绑定开始
   - `[Preview]` - 预览图显示
   - `[Compressed]` - 压缩版本显示
   - `[FullImage]` - 完整大图加载完成
   - `[Target] onResourceReady` - 图片资源就绪

2. 包含性能关键信息：
   - 加载时间（`Loaded in XXXms`）
   - 解码时间（`decodeTime: XXXms`）
   - 图片尺寸和大小
   - 可见范围判断

3. 包含进度条相关信息：
   - 进度条显示/隐藏状态
   - 进度更新过程

### 日志分析示例

查找大图加载慢的问题：

```bash
# 查看图片加载时间
grep "\[FullImage\].*Loaded in" logs/*.log | tail -20

# 查看解码时间
grep "decodeTime:" logs/*.log | tail -20

# 查看可见范围判断
grep "Not in visible range\|In visible range" logs/*.log | tail -20

# 查看进度条状态
grep "progress bar\|circularProgressBar" logs/*.log | tail -20
```

## 注意事项

1. **调试期间**：建议使用 **DEBUG** 级别
2. **正常使用**：建议使用 **INFO** 级别（默认）
3. **性能测试**：可以使用 **WARN** 或 **ERROR** 级别，减少日志开销
4. **日志文件大小**：DEBUG 级别的日志文件会比较大，定期清理
